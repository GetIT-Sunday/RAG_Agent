# 简单实现思路说明

本项目实现了一个本地 RAG 知识库系统，并将知识库检索能力封装成可被 Agent 调用的 MCP tool / MCP server。

## 1. 总体思路

系统分为两条核心链路：

```text
入库链路 Ingestion Pipeline
用户上传 txt / 直接输入文本
  -> 文本清洗
  -> 段落感知切分
  -> 生成 chunk embedding
  -> 保存到 SQLite
```

```text
查询链路 Retrieval Pipeline
Web Search API / Streaming Search API / MCP Tool
  -> 统一调用 SearchService
  -> query embedding
  -> 向量相似度检索
  -> 关键词/BM25-like 检索
  -> hybrid ranking
  -> 按知识库文档聚合结果
```

Web 端和 Agent 端是两个入口，但二者复用同一套 `SearchService`，避免重复实现检索逻辑。

## 2. 知识库实现

知识库使用 FastAPI + SQLite + SQLAlchemy 实现。

主要表：

- `knowledge_documents`：保存知识库标题、原文、来源类型、创建/更新时间。
- `knowledge_chunks`：保存切分后的文本片段、`embedding_json`、chunk 序号。
- `search_logs`：保存查询记录、入口类型、耗时、错误信息等。

支持：

- 知识库新增、查询、编辑、删除。
- 分页列表。
- txt 上传。
- 直接输入文本。
- 真实语料入库脚本。

## 3. RAG 检索实现

文本切分采用 paragraph-aware chunking，而不是简单按固定字符数硬切。这样可以尽量保留文学文本中的段落和场景上下文，例如《故乡》中“少年闰土”的完整场景。

查询时采用 hybrid retrieval：

```text
final_score = 0.75 * vector_score + 0.25 * keyword_score
```

其中：

- 向量检索用于支持语义查询，例如“小孩子”匹配《故乡》。
- 关键词检索用于增强精确词匹配，例如“少年闰土”。
- 文档聚合用于把 chunk 结果还原成知识库文档结果。

如果知识库中没有足够相关内容，系统不会强行返回弱相关文档，而是返回：

```text
不知道，知识库中未找到与该问题足够相关的内容。
```

MCP 工具还会返回 `fallback_to_base_model=true`，提示 Agent 可以使用基础模型继续回答。

## 4. 流式返回实现

流式返回使用 FastAPI `StreamingResponse` + SSE。

查询接口：

```text
GET /api/search/stream?query=春天&top_k=5
```

后端把检索状态、文档标题、片段内容拆成 SSE event 返回，前端逐字显示，模拟 DeepSeek/ChatGPT 文字逐步出现的效果。

## 5. Agent / MCP 实现

本项目选择实现 MCP tool / MCP server，而不是 skill。

MCP server 入口：

```text
run_mcp.py
```

暴露工具：

```text
search_knowledge(query: str, top_k: int = 5)
```

Agent 调用流程：

```text
Agent 收到用户问题
  -> 调用 MCP tool search_knowledge
  -> MCP tool 调用 SearchService
  -> SearchService 查询 SQLite 知识库
  -> 返回结构化结果给 Agent
```

MCP Server 只做工具层封装，不重复实现检索逻辑。

项目提供了：

- `mcp_config.example.json`：Agent MCP 配置示例。
- `examples/mcp_protocol_test.py`：真实 MCP stdio 协议测试。
- `app/mcp/server.py`：MCP server 定义。
- `app/mcp/tools.py`：Agent-friendly 工具返回结构。
